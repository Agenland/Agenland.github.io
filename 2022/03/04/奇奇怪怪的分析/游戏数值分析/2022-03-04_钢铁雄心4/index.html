# 分组

坦克的组件有 9 类:' 底盘 ', ' 模块 ', ' 引擎模块 ', ' 装甲模块 ', ' 悬挂模块 ', ' 炮塔 ', ' 火炮 ', ' 引擎 ', ' 装甲'

其中有以下规矩:

  • ' 底盘 ',' 引擎模块 ', ' 装甲模块 ', ' 悬挂模块 ', ' 炮塔 ', ' 火炮 ' 有且只有一个
  • ' 引擎 ', ' 装甲 ' 是升级，级数从 0~20 中选择
  • ' 模块 ' 是附加的，最多可以加 4 个，也可以不加

其他细节:

  • ' 底盘 ',' 炮塔 ', ' 火炮 ' 可以分为 ' 轻型 ', ' 中型 ', ' 重型 ', ' 超重型 ', ' 现代 '. (现代可以算作重型)
  • 底盘只可以搭配和它同等重量等级或者比它轻的炮塔.
  • 同理，炮塔只可以搭配和它同等重量等级或者比它轻的火炮 (固定型除外)
  • 如果炮塔是固定的，则可以搭配更重一级的火炮，比如固定轻型可以搭配中型火炮但不能搭配重型.
  • ' 模块 ' 中有些组件可以用多次，有些只能用一次或两次 (因底盘类型而异)

# 遗传算法

  • 基因: 9 类组件
  • 变异：基因中的一类组件发生变化 (大部分是直接重新初始化)
  • 交叉：我这里使用的比较特殊，有一方只给出基因中的一类组件，另一方给出剩下的组成新个体
  • 淘汰：我后期调整的是淘汰得分的后 40%
  • 外来入侵：随着时间推移，会有越来越多的全新个体出现，以避免陷入局部最优

# 结果

# 使用遗传算法

目标:

max (' 对人员杀伤 ' + ' 对装甲杀伤 ') / ' 成本'

在 10~40 秒内收敛到 5.7 以上，多试一两次就可以在 10s 内收敛到最优 5.84

而随机生成过了 2 分钟都只能收敛到 5 以下
